# Техническое задание: Разработка Telegram Mini App (TMA) на Vue 3

## 1. Стек технологий и окружение
- **Язык**: JavaScript / TypeScript
- **Фреймворк**: Vue 3 (Composition API), Vite, Tailwind CSS, Pinia, Vue Router.
- **Интеграция**: Telegram WebApp SDK (`@telegram-apps/sdk` или библиотека из скрипта `telegram-web-app.js`).
- **Связь с сервером**: Axios (REST API) + WebSockets (для real-time отслеживания шагов онбординга).

---

## 2. Инициализация и динамическое брендирование

### 2.1. Чтение Telegram Init Data и заголовок Мультитенантности
При монтировании приложения необходимо считать данные авторизации Telegram:
- Получить `window.Telegram.WebApp.initData` (сырая строка, необходимая для проверки подписи бэкендом).
- Извлечь username текущего бота из `window.Telegram.WebApp.initDataUnsafe.receiver.username` (или передавать через Query-параметр/параметр `startapp` при первом запуске).
- Настроить Axios-интерцептор, который автоматически добавляет заголовок `X-Bot-Username: <username_бота>` во все исходящие запросы. Это позволяет бэкенду определять, к какому тенанту (ресторану) относится запрос, и использовать правильный токен для валидации.

### 2.2. Применение динамической темы
После выполнения запроса авторизации бэкенд возвращает настройки брендинга организации (`branding: { design_color, greeting_text, logo_url }`).
- Реализуй Pinia-хранилище `themeStore`.
- Если организация вернула кастомный `design_color` (HEX-код), динамически примени его к приложению: установи CSS-переменную (например, `--color-primary`) на корневом элементе `document.documentElement.style.setProperty('--color-primary', design_color)`.
- Если кастомного цвета нет, используй нативные цвета темы Telegram: `window.Telegram.WebApp.themeParams.button_color` или `bg_color`.
- Вызови методы SDK для покраски статус-бара и хедера в тон темы:
  ```javascript
  window.Telegram.WebApp.setHeaderColor(branding.design_color || 'bg_color');
  window.Telegram.WebApp.setBackgroundColor(branding.design_color || 'bg_color');
  ```

## 3. Бесшовный флоу онбординга (Запрос номера телефона)

Основная бизнес-цель онбординга — получить проверенный номер телефона пользователя через нативный механизм Telegram, исключая ручной ввод и СМС-верификацию.

### 3.1. Алгоритм экрана «Запрос Контакта» (`/onboarding/phone`)
1. Если бэкенд в ответе авторизации вернул `is_onboarded: false` (в профиле отсутствует телефон), роутер перенаправляет пользователя на этот экран.
2. На экране отображается приветствие, логотип ресторана и большая кнопка: **«Поделиться номером телефона»**.
3. По клику на кнопку запускается метод `requestContact()`:
   - **Современный метод:**
     ```javascript
     window.Telegram.WebApp.requestContact((shared) => {
         if (shared) {
             // Пользователь согласился поделиться контактом. Telegram отправил контакт боту.
             // Бот принял его на бэкенде через вебхук и записал в БД.
             // Теперь клиент ждет команду от бэкенда через WebSocket.
             connectWebSocket();
         } else {
             // Обработка отмены
             error.value = "Вы отклонили запрос. Номер телефона обязателен для участия в программе лояльности.";
         }
     });
     ```
   - **Резервный метод (Fallback для старых версий Telegram):**
     Если `requestContact` недоступен в SDK, использовать кнопку `window.Telegram.WebApp.MainButton`:
     ```javascript
     const mainButton = window.Telegram.WebApp.MainButton;
     mainButton.setText("Поделиться контактом");
     mainButton.setParams({ request_contact: true });
     mainButton.show();
     mainButton.onClick(() => {
         connectWebSocket();
         mainButton.hide();
     });
     ```

### 3.2. WebSocket соединение (Ожидание подтверждения)
Параллельно с запросом контакта фронтенд открывает WebSocket по адресу:
`ws://<domain>/ws/loyalty/user_updates/<customer_id>/?token=<JWT_ACCESS_TOKEN>`
- При получении сообщения:
  ```json
  {
    "type": "user_update",
    "message": {
      "event": "phone_updated",
      "phone": "+77011112233"
    }
  }
  ```
- Фронтенд сохраняет полученный телефон в Pinia (`authStore.user.phone`), закрывает WebSocket-соединение и перенаправляет пользователя на Главный экран (`/`).

### 3.3. Резервный поллинг (Fallback Polling)
На случай блокировки WebSockets провайдером или сбоя сети:
- При запуске ожидания контакта запусти `setInterval` с интервалом в 4 секунды.
- Каждые 4 секунды делай запрос `GET /api/loyalty/customer/me/`.
- Если в ответе пришел заполненный номер телефона (`phone` не null) — очисти интервал, закрой сокет и перенаправь пользователя на Главный экран.

## 4. Главный экран Личного Кабинета (`/`)

После успешного онбординга пользователь попадает в личный кабинет лояльности:
1. **Карта Лояльности**:
   - Красивая стеклянная или градиентная карточка, окрашенная в брендовые цвета организации.
   - Отображение текущего баланса баллов гостя.
2. **Генерация QR-кода**:
   - В центре экрана отображается QR-код для сканирования на кассе ресторана.
   - Для генерации использовать библиотеку `qrcode.vue` (или аналог).
   - Содержимое QR-кода: Номер телефона клиента (`+77XXXXXXXXX`) или его `iiko_customer_id`. Это позволит кассиру быстро найти гостя в системе iiko при сканировании.
3. **Wallet Интеграция**:
   - Кнопки-заглушки «Добавить в Apple Wallet» и «Добавить в Google Wallet» для последующей интеграции пропусков Wallet.

## Что ожидается от реализации:
1. Файлы настройки Telegram SDK (`services/telegram.js`).
2. Pinia-хранилище `auth.js` с поддержкой JWT, сохранением профиля и статуса `is_onboarded`.
3. Pinia-хранилище `theme.js` для динамического переключения тем и работы с CSS-переменными.
4. Vue-компоненты:
   - `App.vue` (логика инициализации TMA, авторизации по initData, применение стилей).
   - `PhoneInputView.vue` (экран онбординга с вызовом `requestContact`, WebSocket-клиентом и поллингом).
   - `HomeView.vue` (главный экран с картой лояльности, баллами и QR-кодом).
